## Description

**VULNERABILITY DETAILS**

This bug can be triggered by cros\_camera\_service's mojo interface which can be accessed from arcvm and chrome browser. So this bug may cause arcvm escape and chrome browser privilege escalation.

This bug is heap buffer OOB write bug, attacker can controll the oob write size and content.

The call path is :  

Camera3DeviceOpsDelegate::ProcessCaptureRequest -> CameraDeviceAdapter::ProcessCaptureRequest  

-> CameraDeviceAdapter::RegisterBufferLocked -> CameraDeviceAdapter::RegisterBufferLocked

In override function CameraDeviceAdapter::RegisterBufferLocked, the arguments are all from mojo ipc:

int32\_t CameraDeviceAdapter::RegisterBufferLocked(  

mojom::CameraBufferHandlePtr buffer) {  

return RegisterBufferLocked(buffer->buffer\_id, std::move(buffer->fds),  

buffer->drm\_format, buffer->hal\_pixel\_format,  

buffer->width, buffer->height, buffer->strides,  

buffer->offsets);  

}

int32\_t CameraDeviceAdapter::RegisterBufferLocked(  

uint64\_t buffer\_id,  

std::vector[mojo::ScopedHandle](javascript:void(0);) fds,  

uint32\_t drm\_format,  

mojom::HalPixelFormat hal\_pixel\_format,  

uint32\_t width,  

uint32\_t height,  

const std::vector<uint32\_t>& strides,  

const std::vector<uint32\_t>& offsets) {  

size\_t num\_planes = fds.size(); <------------ fds is from mojo, client can input many fds to make num\_planes big  

std::unique\_ptr<camera\_buffer\_handle\_t> buffer\_handle =  

std::make\_unique<camera\_buffer\_handle\_t>(); <----------- new a camera\_buffer\_handle\_t  

buffer\_handle->base.version = sizeof(buffer\_handle->base);  

buffer\_handle->base.numFds = kCameraBufferHandleNumFds;  

buffer\_handle->base.numInts = kCameraBufferHandleNumInts;

buffer\_handle->magic = kCameraBufferMagic;  

buffer\_handle->buffer\_id = buffer\_id;  

buffer\_handle->drm\_format = drm\_format;  

buffer\_handle->hal\_pixel\_format = static\_cast<uint32\_t>(hal\_pixel\_format);  

buffer\_handle->width = width;  

buffer\_handle->height = height;  

for (size\_t i = 0; i < num\_planes; ++i) { <-------- mojo client can control the num\_planes  

buffer\_handle->fds[i] =  

mojo::UnwrapPlatformHandle(std::move(fds[i])).ReleaseFD(); <------ but the buffer\_handle->fds size is 4, oob write  

buffer\_handle->strides[i] = strides[i]; <------- strides is from mojo ipc, oob write  

buffer\_handle->offsets[i] = offsets[i]; <-------- offsets is from mojo ipc, oob write  

}

//skip...  

}

**VERSION**  

latest code of chromeos platform2/camera

**REPRODUCTION CASE**

1, build chromium browser using attached diff, deploy it to your chromebooks  

2, after deploied, open the built-in camera app, will crash cros\_camera\_service
