## Description

**VULNERABILITY DETAILS**

This bug can be triggered by cros\_camera\_service's mojo interface which can be accessed from arcvm and chrome browser. So this bug may cause arcvm escape and chrome browser privilege escalation.

The vulnerable call path is :  

Camera3DeviceOpsDelegate::ProcessCaptureRequest -> CameraDeviceAdapter::ProcessCaptureRequest ->  

DeserializeCameraMetadata -> add\_camera\_metadata\_entry -> add\_camera\_metadata\_entry\_raw

In function add\_camera\_metadata\_entry\_raw, the arguments tag, type, data, data\_count are from mojo ipc data:

Problem 1: doesn't check if data\_count bigger than the buffer size of data, will cause oob read from data buffer  

Problem 2: on 32 bit userspace chromebooks, big data\_count value can cause integer overflow

static int add\_camera\_metadata\_entry\_raw(camera\_metadata\_t \*dst,  

uint32\_t tag,  

uint8\_t type,  

const void \*data, <---------- data buffer  

size\_t data\_count) { <---------- data\_count may bigger than data buffer size, cause OOB read

```
if (dst == NULL) return ERROR;  
if (dst->entry_count == dst->entry_capacity) return ERROR;  
if (data_count && data == NULL) return ERROR;  

size_t data_bytes =  
        calculate_camera_metadata_entry_data_size(type, data_count);  <---------- use data_count to calculate  
if (data_bytes + dst->data_count > dst->data_capacity) return ERROR;  <-------- may bypass check by integer overflow because sizeof is 32bit on 32bit userspace chromebooks  

size_t data_payload_bytes =  
        data_count \* camera_metadata_type_size[type];  
camera_metadata_buffer_entry_t \*entry = get_entries(dst) + dst->entry_count;  
memset(entry, 0, sizeof(camera_metadata_buffer_entry_t));  
entry->tag = tag;  
entry->type = type;  
entry->count = data_count;  

if (data_bytes == 0) {  
    memcpy(entry->data.value, data,  
            data_payload_bytes);  
} else {  
    entry->data.offset = dst->data_count;  
    memcpy(get_data(dst) + entry->data.offset, data,  
            data_payload_bytes);                    <---------------- OOB write  
    dst->data_count += data_bytes;  
}  
dst->entry_count++;  
dst->flags &= ~FLAG_SORTED;  
assert(validate_camera_metadata_structure(dst, NULL) == OK);  
return OK;  

```

}
