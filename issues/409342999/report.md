## Description

# Steps to reproduce the problem

1. start Chrome
2. run <https://csynth.github.io/csynth/csynth.html>
3. open devtools/console
4. type 'init' and enter
5. devtools finds function init(), but the page crashes
   It may(?) work first time, but will crash on any repeated test after restart

# Problem Description

Alternate related problem (probably same at root)

1. start Chrome
2. run <https://csynth.github.io/csynth/csynth.html>
3. open devtools/console
4. open JS/utils.js, line 4478, just after function init()
5. put a breakpoint
6. restart (F5 or browser)
7. it will break at breakpoint
8. resume, and it will crash the page Error code: STATUS\_ACCESS\_VIOLATION
   Works with chromium 133 Version 133.0.6943.142 (Official Build) (64-bit)
   Fails with Canary 137 Version 137.0.7116.0 (Official Build) canary (64-bit)
   Also fails using VS Code as the debugging host.
   Probably related to but not the same as <https://issues.chromium.org/issues/399002824>
